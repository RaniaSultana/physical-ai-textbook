# Week-by-Week Course Schedule

This textbook can be delivered over a 12-week semester with 2–3 hours of instruction and 4–6 hours of hands-on labs per week.

## Module 1: Robotic Nervous System (Weeks 1–3)

| Week | Topic | Deliverable |
|------|-------|-------------|
| 1 | ROS 2 fundamentals, nodes, and pub-sub | Hello Node (Exercise 1.1) |
| 2 | Topics, services, and launch files | Multi-node system (Exercise 1.4) |
| 3 | Action servers and parameter servers | Custom action server |

## Module 2: Digital Twin & Simulation (Weeks 4–6)

| Week | Topic | Deliverable |
|------|-------|-------------|
| 4 | URDF format and Gazebo setup | Simple arm URDF (Exercise 2.1) |
| 5 | Physics simulation and domain randomization | Force application test (Exercise 2.3) |
| 6 | Debugging and optimization | Tuned simulation parameters |

## Module 3: NVIDIA Isaac & Perception (Weeks 7–8)

| Week | Topic | Deliverable |
|------|-------|-------------|
| 7 | Isaac Sim setup, synthetic data generation | 1000 synthetic images (Exercise 3.4) |
| 8 | Object detection and sensor integration | YOLOv8 detector in simulation |

## Module 4: Vision & Language Agents (Weeks 9–10)

| Week | Topic | Deliverable |
|------|-------|-------------|
| 9 | VLM fundamentals and prompt engineering | Task decomposition agent (Exercise 4.3) |
| 10 | Closed-loop control and safety | Safe VLA system (Exercise 4.5) |

## Capstone Project (Weeks 11–12)

| Week | Task |
|------|------|
| 11 | Final project development and testing |
| 12 | Project presentations and peer review |

## Sample Weekly Schedule (3-hour blocks)

**In-Class (1.5 hours):**
- Lecture on theory and concepts (45 min)
- Live code walkthrough (45 min)

**Hands-on Lab (1.5 hours):**
- Group or pair programming on exercises (1.5 hours)
- Q&A and debugging (ongoing)

**Out-of-Class (4–6 hours):**
- Code assignments and reading (2–3 hours)
- Capstone project work (2–3 hours)

## Grading Rubric (Suggested)

- **Exercises** (40%): One exercise per module, auto-graded or peer-reviewed
- **Participation** (10%): Attendance, lab engagement, forum activity
- **Capstone Project** (50%): Final project demo, code quality, documentation

## Resources

- **Hardware**: Refer to [Hardware Lab](./hardware-lab.md) for robot platform recommendations
- **Software**: All code examples run in a Docker container or Codespace
- **Discussion**: Use GitHub Discussions or a course forum for Q&A
- **Office Hours**: 2 hours per week for 1-on-1 debugging
